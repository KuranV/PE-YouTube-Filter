‚óŹ The cleanest free setup for this is Cloudflare Workers + D1 (their SQLite
  database). Free tier is 100k requests/day and 5GB storage ‚ÄĒ more than enough.

  The idea:

  1. Extension sends reports to your Worker API instead of (or alongside) GitHub
   issues
  2. Worker stores them in D1 ‚ÄĒ tracking how many users reported each channel
  3. When a channel hits a threshold (e.g. 3 reports), the Worker automatically
  creates a GitHub PR to add it to channels.json using the GitHub API with a bot
   token
  4. You review and merge, or set it to auto-merge

  So the flow becomes fully automatic on the user side ‚ÄĒ they just hit "Report",
   done. You only touch it when reviewing the auto-generated PRs.

  Rough API surface you'd need:
  - POST /report ‚ÄĒ receives channel reports from the extension
  - GET /channels ‚ÄĒ optionally serve the channel list from there instead of raw
  GitHub (faster, more control)

  To set it up you'd need:
  - A free Cloudflare account
  - A D1 database (created via Wrangler CLI)
  - A GitHub fine-grained token scoped to just that repo for creating PRs

‚óŹ Done. Here's what to do to get it running:

  1. Install Wrangler and log in
  npm install -g wrangler
  wrangler login

  2. Create the D1 database
  wrangler d1 create pe-reports
  Copy the database_id it prints and paste it into wrangler.toml.

  3. Run the schema
  wrangler d1 execute pe-reports --file=worker/schema.sql

  4. Add your GitHub token as a secret

  Create a fine-grained PAT on GitHub scoped to just PE-YouTube-Filter with
  Contents (read+write) and Pull requests (write) permissions. Then:
  wrangler secret put GITHUB_TOKEN

  5. Deploy
  wrangler deploy

  6. Fill in the worker URL in report.js

  After deploy, Wrangler prints the URL (e.g.
  https://pe-youtube-filter.yourname.workers.dev). Replace
  REPLACE_WITH_YOUR_WORKER_URL in report.js with it.
